# Blacklight Web Designs: Portfolio Specifications

## The "Blacklight" Showcase Standard
Our portfolio is not just a gallery; it's a demonstration of technical supremacy.

### 1. The "Interactive First" Hero
Every portfolio entry must start with a live, interactive fragment of the project's hero section or a key interaction. No static screenshots for heroes.

### 2. The "Maestro Metrics" Panel
Each case study must display a side-by-side comparison:
- **Lighthouse Scores:** (Ours vs. Industry Average)
- **Time to Interactive:** (Ours vs. Competitors)
- **Accessibility Compliance:** (WCAG AA/AAA certification)

### 3. The "Topological Tale"
A brief explanation of why the layout was chosen and which "Safe Harbor" clichés were explicitly avoided.

### 4. Code Snippets (The "Technical Proof")
Highlight a specific "Maestro-level" code fragment (e.g., custom spring-physics animation, optimized data fetching layer, or a radical CSS layout trick).

## Entry Requirements
- **Visuals:** High-contrast, sharp geometry, no pure white backgrounds unless for brutalist effect.
- **Motion:** Staggered reveal of case study sections.
- **Depth:** Grain textures, overlapping layers, and parallax visuals.
